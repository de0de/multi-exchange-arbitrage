# Multi-Exchange Arbitrage — Vision & Business Model

> Этот документ описывает долгосрочное видение проекта. Не является техническим планом — 
> актуальные задачи разработки смотри в PLAN.md.

## Миссия

Создать open-source инструмент для алгоритмической арбитражной торговли, который позволит 
не только автору, но и сообществу пользователей зарабатывать на арбитражных возможностях 
между биржами — по аналогии с Hummingbot, но с собственной моделью монетизации.

## Модель Open-Core

**Бесплатно (open-source, MIT/Apache):**
- Базовая инфраструктура сбора данных (мультибиржевой сбор котировок spot/futures)
- Стратегия **spot-spot арбитраж** — полностью открыта и бесплатна для всех
- Базовый арбитражный калькулятор (адаптация подхода Hummingbot)

**Платно (собственные продукты):**
- Продвинутые стратегии (funding-арбитраж, треугольный арбитраж, DEX-CEX арбитраж и др.)
- Копитрейдинг-модуль с распознаванием сигналов (текст + скриншоты)
- Готовые интеграции с приватными API бирж (управление балансами, исполнение сделок)

## Маркетплейс стратегий

- Пользователи сообщества смогут публиковать и продавать собственные стратегии через 
  встроенный маркетплейс
- Автор проекта получает процент с каждой продажи
- **Требуется до запуска:** механизм модерации/верификации стратегий (backtesting на 
  исторических данных, рейтинги, отзывы) — чтобы защитить покупателей от заведомо убыточных 
  или мошеннических стратегий

## Правовые и этические ограничения (важно помнить)

- **Атрибуция Hummingbot:** проект использует концепции и частично адаптированный код из 
  Hummingbot (Apache 2.0). Обязательно сохранять copyright notice и указывать источник 
  заимствованного кода в комментариях соответствующих файлов
- **Маркетинг без обещаний доходности:** продавать инструмент/код, а не обещание прибыли. 
  Формулировки вида "эта стратегия даёт X% доходности" могут квалифицироваться как 
  инвестиционная консультация в ряде юрисдикций — избегать
- **Копитрейдинг для третьих лиц и управление чужими средствами** — требует юридической 
  консультации по применимому законодательству до реализации. Не начинать без консультации 
  с юристом

## Алгоритмическая торговля (отложено)

Изучение и интеграция полноценных алгоритмических торговых стратегий (не только арбитраж) 
запланированы на более поздний этап, после того как основа проекта (сбор данных, 
арбитражный движок, инфраструктура сообщества) будет стабильна.

## Ориентировочный горизонт

Это видение на 1+ год вперёд. Текущий фокус разработки — см. раздел 5 PLAN.md 
(масштабирование до 5 бирж → перенос на VPS → миграция БД → арбитражный движок).

## Будущее направление: Gamma Exposure (GEX) / опционная аналитика

**Отложено до стабилизации основного арбитражного функционала.**

Идея: изучить и, возможно, интегрировать Gamma Exposure анализ (dealer positioning, gamma flip, 
put/call walls) как дополнительный контекстный сигнал для будущих стратегий — по аналогии с 
сервисами типа ZeroGEX (для SPY/SPX/QQQ) и их аналогами для крипты (CryptoGamma.io, GammaFlip.io).

**Ключевые тезисы:**
- GEX — это НЕ арбитраж в классическом смысле, а структурный/контекстный сигнал для 
  направленной или опционной торговли (другой класс риска, чем текущий spot-spot арбитраж)
- Основной источник ликвидности для крипто-опционов — Deribit API
- Потенциальное применение в будущем: фильтр для арбитражных входов (избегать сделок 
  у сильных gamma-стен), либо отдельная опционная стратегия (volatility arbitrage, 
  basis + gamma)
- Требует серьёзного изучения: формулы Black-Scholes, греки (delta, gamma, theta), 
  агрегация open interest по strikes/expirations
- Найденные open-source примеры (dankbit, gamma_studies, GammaGEX и др.) — рассматривать 
  только как учебный материал для понимания концепции, НЕ как готовый к использованию код: 
  формулы там сильно упрощены, а сами репозитории малы и непроверены

**Когда возвращаться к этой теме:** после того как базовый арбитражный движок (spot-spot, 
spot-futures) стабильно работает и приносит первые результаты. Не смешивать изучение опционов 
с текущим фокусом разработки.
